# UI 交互改进建议 (2026 H1) — 竞品 UI 参考

> **承认不足**:之前的 P0/P1 借鉴主要看了**功能点**(API 端点 / 数据结构 / 平台支持),**没专门看 UI 交互**。这份是补漏 — 基于 135 编辑器 / 壹伴 / AIWriteX / Copy.ai / Jasper 等竞品 UI + Lazyweb 调研 + 我们现状,给 5 个核心视图的改进建议。
>
> **状态**:建议文档,未实现。**用户评审后选 1-2 个高 ROI 的开干**。

---

## 1. AI 中心 (`/ai`) — "左 sidebar + 右 preview" 是行业标准

### 现状
- 左侧功能 tab(摘要/扩写/标题/...) 7 项堆叠
- 主区单栏表单
- 右侧**没有 preview** — 用户必须等生成完才能看

### 借鉴(135 编辑器 / Copy.ai / Manus AI Studio)
- **左 sidebar(功能) + 中 input + 右 live preview** 的三栏布局
- **Streaming 生成** — 边生成边显示(不要等全部完成才返)
- **Diff 视图** — 原文 vs 生成结果并排,逐句高亮改动(Jasper 风格)
- **AI 工具切换不动输入** — 切换"扩写"到"标题"tab,原文不丢
- **快捷键** — `⌘+Enter` 触发生成,`⌘+R` 重新生成

### 具体建议
1. **三栏布局**: 220px(左) + flex(中) + 360px(右 preview)
2. **Streaming output**: 后端 `/ai/expand` 加 `?stream=true` (SSE),前端用 EventSource
3. **历史记录** sidebar: 最近 10 次生成,可一键回填到输入框(Notion AI / Copy.ai 风格)
4. **重新生成按钮**: 每次结果旁 `↻ 换一版` 按钮(不重输 prompt)

### 工作量
- 三栏布局: 1 天
- Streaming: 2-3 天(SSE + 后端 stream 改造 + 前端 EventSource)
- 历史: 1 天
- **合计: 4-5 天, ROI ⭐⭐⭐⭐**

## 2. 内容编辑器 (`/content/edit/:id`) — 借鉴 Notion + 壹伴 + 135

### 现状
- 标题输入框 + Markdown textarea + 预览 tab
- 平台下拉(单选)
- 标签 chip 输入
- AI 助手弹窗(扩写/摘要/改写/标题/标签)
- **没有主题选择**(虽然后端 P0-7 已经支持 default/grace/simple)

### 借鉴
- **Notion** 的 split view 实时预览
- **壹伴** 右侧 toolbar(图床/字数/表情/AI 助手) 常驻
- **135 编辑器** 的左侧模板宫格 + 中心编辑 + 右侧样式

### 具体建议
1. **实时 Markdown 预览** — 输入即预览(split pane 50/50),不要 tab 切换
2. **右侧 AI 助手常驻 sidebar** — 不要"打开弹窗",常驻不挡视线
3. **顶部主题下拉** — `default / grace / simple`(P0-7 已支持但 UI 没暴露)
4. **字数 + 段落数 + 预计阅读时长** footer 状态条
5. **平台勾选多选** — 不是单选(用户想一稿多发)
6. **快捷模板** 左侧抽屉 — 快速套用模板(已经在 `template` 模块有)
7. **自动保存指示** — "上次保存 5 秒前"

### 工作量
- Split preview: 1 天
- 常驻 AI sidebar: 2 天
- 主题选择 + 平台多选: 0.5 天
- 字数状态条 + 自动保存: 1 天
- **合计: 4-5 天, ROI ⭐⭐⭐⭐⭐**(直接提升写作体验)

## 3. 选题雷达 (`/sources`) — 后端 P0-2 已就绪,前端借鉴新榜/易撰

### 现状
- 没有"热榜" tab(API 已有)
- 4 个 tab: 全部/PDF/URL/文本/NotebookLM

### 借鉴
- **新榜小豆芽**: 顶部 banner 滚动播"今日热点 + 24h 涨幅"
- **易撰**: 左侧分类筛选 + 右侧热度趋势图
- **AIWriteX**: 选题卡片大字号 + "AI 改写"按钮显眼

### 具体建议
1. **新增"热榜" tab** — 用 P0-2 API
2. **热榜卡片大字号设计**: 标题(2 行)/ 来源平台徽章 / 热度数字(千/万)/ AI 改写按钮
3. **平台徽章颜色** — 微博红 / 知乎蓝 / 抖音黑 / 小红书红(品牌色)
4. **热度排序** + **一键创建内容** 按钮(走 P0-2 `/api/hot/{id}/create-content`)
5. **每条选题显示已 AI 改写过的"角度预览"** — 用户先看角度再决定要不要

### 工作量
- 4-5 天(后端 API 已就绪,主要做 UI)

## 4. 平台账号管理 (`/platform`) — 借鉴 蚁媒 + 易媒助手

### 现状
- 列表(平台/账号名/状态)
- 新建/编辑/删除 弹窗

### 借鉴
- **蚁媒助手**: 平台 logo 大卡片布局(不是 list), 6 平台一屏看全
- **易媒助手**: 账号状态 + 流量统计 + 一键扫码登录按钮(可点击看到 cookie 路径)
- **Notion**: 卡片 grid + 标签 + 描述

### 具体建议
1. **6 平台卡片 grid**(替换列表)— 每张卡: logo / 状态 / 账号数 / 添加按钮
2. **状态徽章色彩化**: ✅ 活跃(绿) / ⚠️ 待扫码(黄) / ❌ 异常(红)
3. **公众号测试号引导** — 加一个"不知道怎么拿 AppID?"展开说明

### 工作量: 1-2 天

---

## 5. 发布对话框 (`Content/View.vue` 的"发布到公众号") — 借鉴 壹伴

### 现状
- 账号下拉 + 封面预览 + 内联图计数 + 确认按钮
- 弹窗式,一次性

### 借鉴
- **壹伴 / 135**: **多步 wizard** (选账号 → 选主题 → 预览公众号效果 → 确认)
- **公众号助手**:
  - Step 1: 选账号(账号列表 + 状态)
  - Step 2: 选主题(default/grace/simple) + **实时预览公众号效果**
  - Step 3: 确认发布(显示 estimated reach)
- 错误时**就地显示**重试按钮(不要 toast 完就关)

### 具体建议
1. **3 步 wizard**:
   - Step 1: 选账号
   - Step 2: 选主题(3 套 default/grace/simple) + 右侧实时公众号预览(用 P0-7 `/api/content/format`)
   - Step 3: 确认 + 显示结果
2. **失败时**就地重试按钮(保留 dialog)
3. **发布进度可视化** — 4 步串行显示: 图床上传 → 草稿写入 → 派发 → 轮询 (30s 进度条)

### 工作量
- 1 周, ROI ⭐⭐⭐⭐(P0-1 后端已就绪,UI 是最后一公里)
